# 사또 앱 기획서

클로드코드 전달용 문서

## 1. 프로젝트 개요

- 앱명: `사또`
- 앱 한줄 소개: 사용자의 사주 정보를 입력받아 로또 번호를 개인화 추천하고, 간단한 운세 해석과 저장/공유 경험을 제공하는 모바일 앱
- 서비스 성격: 엔터테인먼트 기반 개인화 추천 앱
- 핵심 가치: 완전 랜덤 번호보다 "나에게 맞춘 느낌"을 주는 개인화 경험

`사또`는 `사주`와 `로또`를 결합한 이름으로, 기억하기 쉽고 한국 사용자에게 직관적으로 의미가 전달되는 브랜드를 목표로 한다.

## 2. 기획 배경

기존 로또 번호 추천 앱은 대부분 단순 랜덤 생성기에 가깝고, 사용자가 재방문할 이유가 약하다. `사또`는 사주 정보를 입력받아 번호를 추천함으로써 개인화된 이유를 제공하고, 결과를 해석 콘텐츠와 함께 보여주어 몰입감과 공유 욕구를 높인다.

핵심 방향은 다음과 같다.

- 재미와 감성은 살리되 과장된 당첨 보장은 피한다.
- 입력 후 빠르게 결과를 보여주는 짧은 첫 경험을 설계한다.
- 저장, 히스토리, 공유를 통해 재방문과 바이럴을 만든다.
- 광고와 부분 유료화를 결합한 수익 구조를 만든다.

## 3. 목표 사용자

핵심 타깃은 다음과 같다.

- 20~30대 운세, 테스트, 가벼운 점술 콘텐츠에 익숙한 사용자
- 로또를 투자보다 재미와 기대감으로 즐기는 사용자
- 결과를 캡처하거나 공유하는 데 익숙한 사용자

확장 타깃은 다음과 같다.

- 30~50대 사주와 운세 콘텐츠에 친숙한 사용자
- 단순 랜덤이 아닌 "설명 있는 번호 추천"을 원하는 기존 로또 사용자

## 4. 서비스 목표

이 앱의 1차 목표는 번호 추천 경험 자체를 재미있고 반복 가능하게 만드는 것이다.

구체 목표:

- 설치 후 1분 이내 첫 번호 추천 완료
- 사용자가 결과를 저장하거나 공유하도록 유도
- 매주 로또 회차 기준으로 재방문 유도
- 무료 사용자도 충분히 즐기게 하되, 추가 조합과 상세 해석은 유료 또는 광고 보상으로 전환

## 5. 제품 포지셔닝

`사또`는 다음 세 가지 균형을 유지한다.

- 재미: 너무 무겁지 않고, 결과를 보는 순간이 즐거워야 한다.
- 그럴듯함: 사주 기반 해석이 어느 정도 납득감 있게 느껴져야 한다.
- 안전성: 당첨 보장이나 투자 권유처럼 보이지 않도록 해야 한다.

한 줄 포지셔닝:

`내 사주 흐름으로 뽑는, 가장 개인적인 로또 번호`

## 6. 핵심 사용자 경험

사용자 플로우는 아래와 같다.

1. 사용자가 생년월일, 성별, 태어난 시간, 양력/음력 정보를 입력한다.
2. 앱이 사주 기반 가중치와 랜덤성을 섞어 로또 번호를 추천한다.
3. 추천 번호와 함께 오늘의 복권운, 해석 문구, 추천 시간대 등을 보여준다.
4. 사용자는 결과를 저장하거나 공유한다.
5. 다음 회차에 푸시나 홈 피드로 다시 방문한다.

## 7. MVP 범위

1차 출시에서는 아래 기능까지 포함한다.

- 사주 정보 입력
- 로또 번호 추천
- 조합별 간단 해석
- 오늘의 복권운 표시
- 결과 저장
- 추천 히스토리
- 이미지 카드 공유
- 광고 보상형 추가 조합 열기
- 부분 유료 상품 연동

1차에서 제외한다.

- 커뮤니티
- 실제 복권 구매 연동
- 당첨 인증 피드
- 전문가 상담
- 지나치게 복잡한 사주 풀이

## 8. 핵심 기능 상세

### 8-1. 온보딩 및 입력

입력 항목:

- 생년월일
- 성별
- 태어난 시간
- 양력/음력 선택

선택 항목:

- 출생 지역

입력 UX 원칙:

- 단계형 입력으로 부담을 낮춘다.
- 태어난 시간을 모르는 사용자를 위해 "모름" 또는 "대략" 옵션을 둔다.
- 회원가입 없이 먼저 결과를 보게 한다.
- 저장 또는 기기 변경 동기화 시점에만 로그인 유도 가능

### 8-2. 번호 추천 엔진

중요한 원칙은 `사주 기반 + 변주 가능한 랜덤성`이다.

권장 구조:

- 입력된 사주 정보로 숫자군 선호 가중치 계산
- 구간별 숫자, 홀짝 비율, 끝수 패턴, 연속수 여부 등에 가중치 적용
- 완전 고정 결과가 되지 않도록 랜덤 요소 혼합
- 동일 사용자가 반복 사용해도 일정한 결은 유지하되, 결과는 매회 달라질 수 있게 설계

무료 제공:

- 기본 1조합

광고 또는 유료 제공:

- 추가 2~4조합

조합 라벨 예시:

- `재물 기회형`
- `안정 상승형`
- `직감 몰림형`
- `균형 흐름형`

### 8-3. 해석 콘텐츠

결과 화면에는 번호만이 아니라 해석 요소를 함께 제공한다.

필수 요소:

- 추천 번호 6개
- 보너스 추천 숫자 1개
- 오늘의 복권운 점수
- 추천 이유 2~3줄
- 피해야 할 기운 또는 보완 포인트 1줄
- 추천 구매 시간대
- 행운 컬러 또는 행운 키워드

카피 톤 예시:

- 오늘은 강한 한방보다 균형 있는 조합이 잘 맞는 흐름입니다.
- 무리한 기대보다 자연스럽게 들어오는 운을 잡는 날입니다.
- 직감형 숫자보다 안정형 숫자대에서 흐름이 살아납니다.

### 8-4. 저장 및 히스토리

사용자가 기록하고 다시 보게 만드는 기능이다.

포함 요소:

- 날짜별 추천 번호 저장
- 회차별 저장
- 지난 추천 다시 보기
- 내가 자주 받은 숫자 확인

향후 확장 가능:

- 지난주 추천과 이번주 추천 비교
- 최근 4주 운 흐름 시각화

### 8-5. 공유 기능

공유는 바이럴 핵심 기능이다.

공유 포맷:

- 인스타 스토리형 세로 카드
- 카카오톡 공유용 이미지 카드

공유 카드 포함 요소:

- 추천 번호
- 오늘의 복권운
- 해석 한 줄
- 앱명 `사또`
- 앱 워터마크

## 9. 정보 구조 및 화면 구성

필수 화면:

- 스플래시
- 온보딩 및 정보 입력
- 홈
- 결과 화면
- 히스토리
- 공유 카드 생성 화면
- 결제 및 광고 보상 팝업
- 설정
- 개인정보 및 면책 고지 화면

### 9-1. 홈 화면

핵심 구성:

- 오늘의 복권운
- 빠른 번호 추천 버튼
- 지난 추천 다시 보기
- 이번 주 행운 키워드
- 프리미엄 조합 열기 CTA

### 9-2. 결과 화면

핵심 구성:

- 메인 번호 카드
- 조합별 탭 또는 카드 스와이프
- 운세 해석 블록
- 저장 버튼
- 공유 버튼
- 추가 조합 열기 버튼

## 10. 수익화 전략

권장 수익화 구조는 `무료 + 광고 + 부분 유료` 조합이다.

무료 영역:

- 기본 추천 1조합
- 기본 해석

광고 보상 영역:

- 추가 조합 열기
- 재추천
- 프리미엄 카드 템플릿 열기

부분 유료 상품 예시:

- 오늘의 프리미엄 번호팩
- 이번 주 5조합 패키지
- 월간 재물운 리포트
- 광고 제거
- 상세 사주 해석

2차 확장 시 구독 모델 검토 가능:

- 주간 자동 추천
- 회차 알림 강화
- 프리미엄 해석 무제한 열람

## 11. 리텐션 설계

재방문을 만들 장치:

- 로또 회차 기준 푸시 알림
- "이번 주 복권운이 열렸어요" 형태의 리마인드
- 저장한 사용자 대상 새 조합 안내
- 연속 방문 스탬프 또는 배지

기본 루프:

1. 번호 추천
2. 해석 확인
3. 저장 또는 공유
4. 다음 회차 알림
5. 재방문 및 비교

## 12. 톤앤매너 및 디자인 방향

브랜드 키워드:

- 현대적
- 신비로운
- 세련된
- 가볍지만 촌스럽지 않은

디자인 원칙:

- 지나치게 전통적이거나 올드한 점술 UI는 피한다.
- 결과 카드는 감성적이되 숫자 가독성이 최우선이다.
- 컬러는 `딥네이비`, `골드 포인트`, `크림/아이보리` 계열 중심을 추천한다.
- 보라색 일변도는 피하고, 브랜드 차별성을 만든다.

## 13. 법적 및 정책 유의사항

중요: 이 앱은 `당첨 보장`, `확률 상승 보장`, `수익 보장`처럼 해석될 수 있는 표현을 사용하면 안 된다.

반드시 포함할 문구:

- 본 서비스는 오락 및 참고용 콘텐츠입니다.
- 특정 당첨 결과를 보장하지 않습니다.
- 실제 복권 구매 및 사용 판단의 책임은 사용자 본인에게 있습니다.

유의사항:

- 실제 금전 베팅 기능은 넣지 않는다.
- 복권 구매 대행 또는 자동 구매 연동은 하지 않는다.
- 미성년자 대상 과도한 사행성 표현을 피한다.
- 스토어 심사 시 갬블링 오인 가능성을 낮추기 위해 엔터테인먼트 성격을 명확히 한다.

참고 링크:

- Google Play 도박/갬블링 정책: https://support.google.com/googleplay/android-developer/answer/9878810

- 개인정보처리방침과 삭제 요청 기능을 준비한다.

구현은 아래 순서로 진행하는 것을 권장한다.

1. 사주 정보 입력 플로우 구현
2. 번호 추천 엔진 구조 설계
3. 결과 화면 카드 UI 구현
4. 저장 및 히스토리 기능 구현
5. 공유 카드 생성 기능 구현
6. 광고 보상 및 부분 유료 연동
7. 푸시 및 재방문 장치 구현

## 17. 제품 원칙

- 입력은 최소화하고 결과는 빠르게 보여준다.
- 설명은 길지 않게, 짧고 그럴듯하게 제공한다.
- 개인화처럼 느껴지되 결과가 고정되지 않도록 한다.
- 저장과 공유를 자연스럽게 유도한다.
- 과장 표현보다 감성, 몰입감, 반복 사용성을 설계한다.
- 정책 위반 가능성이 있는 카피는 초기에 차단한다.

## 18. 최종 요구사항 요약

`사또`는 사용자의 사주 정보를 입력받아 로또 번호를 개인화 추천하고, 운세 해석, 저장, 공유 경험까지 제공하는 모바일 앱이다. MVP는 엔터테인먼트 기반 서비스로 설계하며, 당첨 보장을 암시하지 않는 카피와 정책을 적용한다.
